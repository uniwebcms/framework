## Summary

These diagrams illustrate the core architectural principles of Uniweb:

1. **üèóÔ∏è Boundary-based design**: Clear separation between exposed (content-facing) and internal (developer) components
2. **üîÑ Dual interfaces**: Traditional code interfaces for internal development, declarative interfaces for content creators
3. **üìã Schema as contract**: Component schemas define the content-facing API and enable validation and tooling
4. **üéØ Single source of truth**: One Foundation serves many sites with controlled update propagation
5. **üåâ Two-world bridge**: Connects the developer ecosystem (React, npm) with the content creator ecosystem (markdown, visual editors)
6. **üìà Progressive complexity**: Supports everything from simple single-site projects to enterprise design systems

**The Foundation is the bridge** that connects these concepts, providing a content-facing interface while maintaining standard React development practices internally.
